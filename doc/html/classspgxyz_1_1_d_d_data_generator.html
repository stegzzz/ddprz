generate the responses using params and Ds

n "trials" are generated. Each trial is randomly determined to use one of a or A as the larger later outcome. The default values of a and A are 820 and 1000 respectively to mirror the values used in ddgem in search of the PRZ project. After setting the value of LL (to a or A) a value for SS is randomly chosen to be some value (0 < SS \leq LL) from among (as) evenly spaced values. The delay to the larger reward D is chosen at random from Ds. By default the delay to the smaller reward d is fixed at zero but can include front end delay.

If a front end delay is included then LL is always set to A and SS is always set to a. D is chosen randomly from Ds and d is then randomly chosen to be some value (0 \leq d < D) from among (ds) evenly spaced values.

get a value below max, the number of values below the max is defined by nstep

one of nstep values is returned at random. If incZero==true the values are chosen from the range max-i*szstep with i in [1, 2, ..nstep]. If incZero==false the range is max-i*szstep + max/nstep with i in [1, 2, ..nstep]. szstep=max/nstep in both cases.

For example, with incZero==true, max=50 and nstep=5 gives possible values 0, 10, 20, 30, and 40 formed by 50-1*szstep, 50-2*szstep, .. 50-5*szstep and szstep=50/5. With incZero==false we have 10, 20, 30, 40, and 50 as possible values.

uses dice
